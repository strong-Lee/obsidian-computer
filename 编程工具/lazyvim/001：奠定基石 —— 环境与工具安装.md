	这一步的目标是安装所有 Neovim 正常工作所需的“外部依赖”。我们将全部通过 macOS 的包管理器 Homebrew 完成，保持系统整洁。

### 步骤 1：安装核心工具 (通过 Homebrew)

	打开你的终端 (例如 WezTerm)，复制并执行以下命令。

> [!TIP]  
如果你还未安装 Homebrew，请先访问 [brew.sh](https://www.google.com/url?sa=E&q=https%3A%2F%2Fbrew.sh) 官网，根据指示安装。它是 macOS 上的必备神器。

```bash
brew update               # 升级brew自己
brew upgrade              # 升级使用brew安装的所有的软件
brew doctor               # 让 brew 自我诊断，它会给出很多有用的建议
brew cleanup              # 清理旧缓存和旧版本，避免干扰

# --- 核心编辑器与版本控制 ---
brew install neovim       # 我们的一切都围绕它，--version，确保版本 >= 0.9.0
brew install git          # macOS 自带的可能版本较旧，brew 版更新更及时
brew install lazygit      # 一个神奇的 Git 终端图形界面，稍后会深度集成

# --- 增强 Neovim 搜索能力的“双引擎” ---
# 它们是核心插件 Telescope 的依赖，用于实现闪电般的文件和内容搜索
brew install ripgrep      # rg: 用于在项目中递归搜索文本内容，比 grep 快得多
brew install fd           # fd: 用于查找文件名和目录，比 find 更快、更直观

# --- PHP 生态系统 ---
brew install php          # 安装 最新最稳定的PHP解释器
brwe tap shivammathur/php # 添加包含所有 PHP 版本的仓库 (tap)
brew install shivammathur/php/php@7.4  # 精准安装php7.4解释器
# 链接到系统，让它成为默认的 php 版本
brew link --overwrite --force shivammathur/php/php@7.4
# 验证一下 PHP 版本 
php -v                    # 输出应该包含 7.4.x
brew install composer     # PHP 的包管理器，用于管理项目依赖和自动加载
brew install node         # 许多 LSP 服务器、格式化器等开发工具都是用 Node.js
```

### 步骤 2：安装 Nerd Font (视觉灵魂)

> [!WARNING]  
> **此步至关重要！**  
> LazyVim 的界面（如文件树图标 、LSP 提示图标 💡）严重依赖这些特殊字体。没有它，你会看到大量的乱码或方框 □。
> 推荐字体：FiraCode Nerd Font 或 JetBrainsMono Nerd Font

```bash
# JetBrainsMono Nerd Font 是公认的顶级编程字体，清晰易读，强烈推荐
brew install --cask font-jetbrains-mono-nerd-font

# (可选) 验证一下是否安装成功，此命令会列出安装的文件
fc-list | grep "JetBrainsMono Nerd"
```

#### B. 配置 Fish Shell 的 PATH

> [!NOTE]  
> **什么是 PATH？**  
> PATH 是一个环境变量，它告诉你的 Shell（比如 Fish 或 Zsh）去哪些目录里寻找你输入的命令。当我们通过 Composer 全局安装工具时（例如 intelephense-ls），需要把它的安装目录 ~/.composer/vendor/bin 添加到 PATH 中，这样系统才能找到并执行它。

	在 Fish Shell 中执行以下命令，它会永久地将 Composer 的 bin 目录添加到你的 PATH 中：


```bash
fish_add_path ~/.composer/vendor/bin
```

保存退出后，重启终端或执行 source ~/.config/fish/[[fish_config.fish]]
