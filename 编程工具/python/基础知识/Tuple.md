## 1. 基础常用
1. **多返回值：** 函数 return a, b 本质是返回元组。
2. **创建单元素元组：** 必须加逗号 (1,)。
3. **多变量赋值：** x, y = 10, 20。
4. **不可变性利用：** 作为字典的 Key（列表不行）。
5. **字符串格式化：** "Name: %s" % ("Alice",) 老式格式化需要元组。
6. **转换为列表：** list(tup)。
7. **嵌套元组：** ((1, 2), (3, 4)) 模拟矩阵。
8. **重复运算：** (1, 2) * 3 得到 (1, 2, 1, 2, 1, 2)。
9. **比较运算：** (1, 2) < (1, 3) 逐位比较。
10. **获取长度：** len(tup)。
## 2. 进阶高级
1. **namedtuple：** from collections import namedtuple，像对象一样访问 p.x。
2. **变量交换：** a, b = b, a (底层利用元组解包)。
3. **星号解包：** a, *rest = (1, 2, 3, 4) -> rest 是 [2, 3, 4]。
4. **忽略元素：** _, b, _ = (1, 2, 3) 使用 _ 占位。
5. **作为集合元素：** 存储坐标点 set([(1,1), (2,2)])。
6. **生成器表达式转元组：** tuple(x**2 for x in range(5))。
7. **count 和 index：** 元组虽然不可变，但也支持这两个查询方法。
8. **元组内的可变对象：** t = ([],)，t[0].append(1) 是合法的（元组只管引用地址不变）。
9. **内存优于列表：** 存储相同数据，元组通常占用内存更小。
10. **__slots__ 优化：** 在类中使用 tuple 结构代替 __dict__ 节省内存。

## 3. 黑客技巧
1. **CPython 小元组缓存：** Python 会缓存长度为 0-20 的空闲元组，创建开销极低。
2. **单例空元组：** () 在内存中永远只有一个实例。
3. **+= 的陷阱：** t = ([],); t[0] += [1] 会报错 TypeError，但数据其实**追加成功了**（这是 Python 面试经典题）。
4. **哈希计算：** 只有当元组内所有元素都可哈希时，元组才可哈希。
5. **元组解包作为参数：** func(*args) 将元组炸开传给函数。
6. **利用 yield from：** 递归展开嵌套元组。
7. **构建 DSL：** 利用元组结构表示抽象语法树 (AST)。
8. **只读数据保护：** 将配置项强制转为元组，防止运行时被意外修改。
9. **字节码差异：** 定义 t = (1, 2) 使用 LOAD_CONST（快），定义列表使用 BUILD_LIST（慢）。
10. **相对导入技巧：** from . import (a, b, c) 利用元组括号换行。
